Authentication & Authorization

OAuth 2.0 Overview​

OAuth 2.0 is a widely used framework for authorization, allowing third-party applications to gain limited access to user resources on a server. It enables users to authorize applications without sharing their credentials, enhancing security and user experience. Our SSO API utilizes OAuth 2.0 to manage authentication and authorization processes, facilitating single sign-on (SSO) and secure API access.

Supported Grant Types​

Our API supports the following OAuth 2.0 grant types:

  1. Authorization Code Grant:

    • Purpose: This flow is used to obtain an authorization code which can then be exchanged for an access token. It is ideal for applications where the client secret can be kept confidential (e.g., server-side applications).
    • Flow:
      1. The client application redirects the user to the authorization server.
      2. The user authenticates and authorizes the application.
      3. The authorization server redirects the user back to the client with an authorization code.
      4. The client exchanges the authorization code for an access token.

  2. Client Credentials Grant:

    • Purpose: This flow is used for machine-to-machine communication where no user context is involved. The client authenticates using its own credentials to obtain an access token.
    • Flow:
      1. The client sends a request to the token endpoint with its client credentials.
      2. The authorization server responds with an access token.

Scopes & Permissions​

Scopes define the level of access that the application is requesting from the user. When the user authorizes the application, they are informed about the permissions requested. Scopes can be used to limit access to specific resources or actions.

Example Scopes:

  • read: Access to read user profile information. (integrated)
  • write: Access to modify user data. (in development)

When requesting authorization, you can specify the scopes needed by including them in the scope parameter of the authorization request.

Security Considerations​

  1. Secure Communication:

    • Always use HTTPS to ensure that all data transmitted between the client, authorization server, and resource server is encrypted.
    • Avoid transmitting sensitive information over insecure channels.

  2. Client Secret Management:

    • Keep client secrets confidential. They should not be exposed in client-side code or public repositories.
    • Use secure methods to store and manage client secrets on the server.

  3. Token Storage:

    • Store access tokens securely and avoid exposing them in client-side code or logs.
    • Implement secure storage mechanisms for tokens and ensure they are transmitted securely.

  4. Token Expiration & Refresh:

    • Implement token expiration and refresh mechanisms to manage long-lived sessions securely.
    • Use refresh tokens to obtain new access tokens without requiring the user to re-authenticate.

Request and Response Format​

Authorization Code Grant:​

  • Authorization Request:

    Endpoint: /oauth/authorize
    Method: GET
    Parameters:

    • response_type (required): Should be code.
    • client_id (required): The ID of the client requesting authorization.
    • redirect_uri (required): The URI to redirect the user after authorization.
    • state (optional): An opaque value used to maintain state between the request and callback.

  • Token Request:

    Endpoint: /oauth/token
    Method: POST
    Parameters:

    • grant_type (required): Should be authorization_code.
    • code (required): The authorization code received from the authorization server.
    • redirect_uri (required): The URI used in the authorization request.
    • client_id (required): The client ID.
    • client_secret (required): The client secret.

Client Credentials Grant:​

  • Token Request:

    Endpoint: /oauth/token
    Method: POST
    Parameters:

    • grant_type (required): Should be client_credentials.
    • client_id (required): The client ID.
    • client_secret (required): The client secret.

Example Scenarios​

Authorization Code Grant:​

  1. Request Authorization Code:

    • URL: https://id.lineardev.net/oauth/authorize?response_type=code&client_id=your_client_id&redirect_uri=your_redirect_uri
    • Response: Redirects to redirect_uri with code and state parameters.

  2. Exchange Code for Token:

    • URL: /oauth/token
    • Method: POST
    • Body:
{
  "grant_type": "authorization_code",
  "code": "authorization_code_from_response",
  "redirect_uri": "your_redirect_uri",
  "client_id": "your_client_id",
  "client_secret": "your_client_secret"
}

Client Credentials Grant:​

  1. Request Token:
    • URL: /oauth/token
    • Method: POST
    • Body:
{
  "grant_type": "client_credentials",
  "client_id": "your_client_id",
  "client_secret": "your_client_secret"
}

By understanding these authentication and authorization processes, you can effectively integrate and utilize our OAuth 2.0 SSO API for secure and seamless user access across your applications.
